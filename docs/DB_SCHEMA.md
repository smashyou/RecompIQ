## Conventions

- All tables: `id uuid primary key default gen_random_uuid()`
- All tables: `created_at timestamptz not null default now()`
- All user-scoped tables: `user_id uuid not null references auth.users(id) on delete cascade`
- All user-scoped tables: **RLS enabled** with policy `auth.uid() = user_id`.
- Time columns: `timestamptz`. Date-only columns: `date`.
- Soft delete: `deleted_at timestamptz` where retention matters.
- Demo flag: `is_demo boolean default false` on user-touchable tables.

## RLS pattern (every user-scoped table)

```sql
alter table <t> enable row level security;

create policy <t>_select on <t> for select
  using (auth.uid() = user_id);
create policy <t>_insert on <t> for insert
  with check (auth.uid() = user_id);
create policy <t>_update on <t> for update
  using (auth.uid() = user_id) with check (auth.uid() = user_id);
create policy <t>_delete on <t> for delete
  using (auth.uid() = user_id);
```

`compounds` and `peptide_kb` are read-only-public (RLS: `select` to `authenticated`,
no `insert/update/delete` policies â†’ blocked).

## Triggers

- `audit_log` trigger on `peptide_doses`, `health_metrics_*`, `ai_messages`,
  `safety_alerts`: insert a diff row on every mutation.
- `set_updated_at` trigger on every table with `updated_at`.
- `weights_after_insert` â†’ enqueue projection recompute (via `pg_notify` â†’
  Vercel cron picks up, or in-process call from the API handler).
